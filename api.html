API Reference

The VibeCaaS API is a RESTful API that allows you to programmatically manage your containerized applications. All API requests must be authenticated using your API key.

Include your API key in the Authorization header:

Deployments

GET /api/v1/deployments

List all deployments in your account.

Query Parameters

Parameter Type Description
limit integer Number of results to return (optional)
offset integer Offset for pagination (optional)
status string Filter by status: running, stopped, failed (optional)

Response Example

{
  "deployments": [
    {
      "id": "dep_123abc",
      "name": "my-app",
      "status": "running",
      "created_at": "2025-01-15T10:00:00Z",
      "repository": "username/repo",
      "url": "https://my-app.vibecaas.com"
    }
  ],
  "total": 1
}
POST /api/v1/deployments

Create a new deployment from a GitHub repository or Docker image.

Request Body

Field Type Description
name string Deployment name (required)
source object Source configuration (required)
environment object Environment variables (optional)
resources object Resource limits (optional)

Request Example

{
  "name": "my-app",
  "source": {
    "type": "github",
    "repository": "username/repo",
    "branch": "main"
  },
  "environment": {
    "NODE_ENV": "production",
    "PORT": "3000"
  },
  "resources": {
    "cpu": "1",
    "memory": "512Mi"
  }
}
DELETE /api/v1/deployments/{id}

Delete a deployment and all associated resources.

Response Example

{
  "message": "Deployment deleted successfully",
  "id": "dep_123abc"
}

Containers

GET /api/v1/containers/{id}/logs

Get logs from a container.

Query Parameters

Parameter Type Description
lines integer Number of lines to return (optional, default: 100)
follow boolean Stream logs in real-time (optional)
POST /api/v1/containers/{id}/restart

Restart a container.

Response Example

{
  "message": "Container restarted successfully",
  "id": "cnt_456def",
  "status": "running"
}
PUT /api/v1/containers/{id}/scale

Scale container replicas.

Request Body

Field Type Description
replicas integer Number of replicas (required)

Metrics

GET /api/v1/metrics/{deployment_id}

Get real-time metrics for a deployment.

Query Parameters

Parameter Type Description
period string Time period: 1h, 24h, 7d, 30d (optional, default: 1h)
metrics array Specific metrics to return (optional)

Response Example

{
  "deployment_id": "dep_123abc",
  "period": "1h",
  "metrics": {
    "cpu": {
      "current": 45.2,
      "average": 38.5,
      "max": 78.3,
      "unit": "percent"
    },
    "memory": {
      "current": 256,
      "average": 245,
      "max": 412,
      "unit": "MB"
    },
    "requests": {
      "total": 15234,
      "rate": 42.3,
      "unit": "req/s"
    }
  }
}

Rate Limits

API requests are rate limited based on your plan:

  • Free: 100 requests per hour
  • Professional: 1,000 requests per hour
  • Enterprise: Unlimited requests

Rate limit information is included in response headers:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1642248000

Error Codes

The API uses standard HTTP status codes:

Code Description
200 Success
201 Created
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API key
403 Forbidden - Access denied
404 Not Found - Resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error
